Steward Bank Freelancer Project

The brief with notes / explanitions.

  1. Capture personal details (Name, surname, ID number, phone number, address and card number used). This has been expanded to include a login name and password, email as well as birthdate. A combination of javascript form validation and Django form validation ensure all fields are entered correctly. Of special note is the ID format.
  2. Capture KYC images (customer’s headshot, ID, Proof of Residence). As this is done on the web app, i have stuck to simply uploading images, limited to .jpg, .png and .pdf format.
  3. Capture the customer’s digital signature (customer should sign on the device screen). Again for the web app, this is limited to uploading a .jpg, .png or .pdf.
  4. Lock user login credentials: the application should not allow multiple login: If user logs onto another device the previous one should be logged out. Currently, if the user is logged in on one device and tries to log in on a second. The second device is blocked. The customer is automatically logged out after 5 minutes of inactivity.
  5. Create an audit report to show the time the account was opened, who opened it, total number of accounts opened by specific agent. To be done in dummy backend
  6. Send SMS notification to customer when account is successfully opened, showing their account number. I have intrgrated with https://www.twilio.com. As there is no credit, the sms's are not sent, but the intregration is in place.
  7. Create a dummy backend to support this process.
